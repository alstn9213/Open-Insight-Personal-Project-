# 파이썬 관련 공부

`if __name__ == "__main__":` 구문은 파이썬에서 **이 파일이 메인 프로그램으로 직접 실행되었을 때만 아래 코드를 실행하라**는 뜻입니다.

`__name__`이라는 변수를 직접 만들지 않더라도 동작하는데, 그 이유는, 이것이 **파이썬이 내부적으로 미리 만들어둔 '특별한 변수(Built-in Variable)'**이기 때문입니다.


### 1\. `__name__` 변수의 정체 (명찰)

파이썬 파일(모듈)은 실행될 때 자기 자신의 '이름표(명찰)'를 답니다. 이 명찰에 적힌 이름이 바로 `__name__` 변수에 저장됩니다.

  * **직접 실행할 때 (`python init_data.py`):**
    파이썬은 "어? 네가 주인공(메인)이네?" 하고 명찰에 \*\*`"__main__"`\*\*이라고 적어줍니다.
  * **다른 파일에서 불러올 때 (`import init_data`):**
    파이썬은 "너는 부품으로 쓰이는구나. 원래 파일 이름으로 적어줄게." 하고 명찰에 \*\*`"init_data"`\*\*라고 적어줍니다.

### 2\. 저 조건문을 쓰는 이유

만약 `init_data.py` 안에 `init_basic_data()` 함수를 실행하는 코드가 그냥 들어있다면 어떻게 될까요?

```python
# 만약 조건문이 없다면...
def init_basic_data():
    print("데이터 적재!")

init_basic_data() # <- 이 줄이 문제!
```

나중에 다른 파일(예: `main.py`)에서 `import init_data`라고 쓰기만 해도, `init_data.py`가 로딩되면서 **`init_basic_data()`가 자동으로 실행되어 버립니다.** 나는 그냥 함수만 가져다 쓰고 싶은데, 원치 않게 DB에 데이터가 또 들어가거나 에러가 날 수 있겠죠.

그래서 **내가 직접 실행 버튼을 눌렀을 때만 실행하고, 남이 나를 불러다 쓸 때는 가만히 있어!** 라고 보호막을 치는 것이 바로 `if __name__ == "__main__":` 입니다.

### 3\. 요약

  * **`__name__`**: 파이썬이 파일마다 자동으로 붙여주는 이름표 변수입니다.
  * **`"__main__"`**: "내가 지금 실행의 주체(메인)"라는 뜻의 값입니다.
  * **결론**: 이 코드는 **안전장치**입니다. 이 파일이 **라이브러리(부품)로 쓰일 때는 실행되지 않도록 막아줍니다.**
