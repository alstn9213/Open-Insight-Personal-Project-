# 개발일지
- 초기 기획에선 api로 업종 별 매출액과 폐업률 정보를 받아오려했으나 그런 민감 정보들은 대부분 유료라, 지역내 동일 업종 수와 유동인구를 바탕으로 사이트 내에서 자체적으로 매출액과 폐업률을 계산하기로 결정 -> 이후 자체 알고리즘으로 계산한 정보들은 의미있는 정보로서의 가치가 떨어진다는 판단하에 아예 매출과 폐업과 관련된 정보를 삭제하기로 결정. 결국, 방향을 선회하여 프로젝트 주제를 `공공데이터를 활용하여 **상권 밀도, 유동인구, 경쟁 현황**을 시각화하고, 예비 창업자의 합리적인 의사결정을 돕는 분석 대시보드` 개발로 최종 결정.
- 한국 내 모든 지역의 유동인구와 업종 수를 불러오려했으나 전국 단위로 통일된 데이터 없어 자료가 제일 풍부한 서울시만 한정적으로 정보를 제공하는 것을 목표로 개발 중.

## 구현 목록
- redis로 데이터 서치 최적화
- docker
- swagger

## redis 실행하는 docker 명령어

```bash
docker run -d -p 6379:6379 --name my-redis redis # 처음 연결
docker start my-redis # 이후에는 이 명령어 사용
```

## 외부 데이터 출처
- 공공데이터 포탈에서 api로 서울시 내의 업종별 점포 수를 알아옴
- 공공데이터 포탈에서 업종별 코드를 csv파일로 받아와 파이썬과 연결해서 업종별 카테고리를 만듬
- 서울 열린데이터광장에서 생활인구 api를 받아와 서울시의 유동인구 조사
- 깃허브에 올라와있는 한국 geojson 파일을 다운받아 파이썬과 연결
- 카카오 맵 api 사용

# 추후 예정

## 구현해야하는 것들

* **Auth2:** 카카오, 구글, 네이버로 로그인 구현

* **테스트 코드 작성** 

## 고민중

* **시뮬레이션 고도화:** 공공데이터 포털의 다른 데이터(예: 주거 인구, 직장 인구 등)를 추가로 수집하여 보정 계수를 정교화하거나, 데이터가 없는 지역은 UI에서 '데이터 부족'으로 명시하는 걸 고민중.

`지금 회원이 할 수 있는 기능이 없어서 뭘 넣을 지 고민중임`
* **즐겨찾기:** 사용자가 관심있는 상권이나 업종을 즐겨찾기 할 수 있는 기능.
